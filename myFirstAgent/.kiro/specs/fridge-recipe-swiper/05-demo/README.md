# 阶段 5: 前端原型 (Frontend Demo)

## 文件说明

| 文件 | 说明 |
|------|------|
| `index.html` | 响应式原型，支持 PC 和移动端 |

## 如何查看

### 方法 1: 直接打开
双击 `index.html` 文件，在浏览器中打开。

### 方法 2: 本地服务器
```bash
cd .kiro/specs/fridge-recipe-swiper/05-demo
npx serve .
```

### 方法 3: VS Code Live Server
使用 VS Code 的 Live Server 插件打开。

## 原型功能演示

### 页面流程

```
1. 上传页面      → 点击上传区域模拟上传照片
      ↓
2. 食材识别     → 显示识别结果，可增删食材
      ↓
3. 加载动画     → 模拟 AI 生成过程
      ↓
4. 滑动选择     → 左滑跳过，右滑收藏
      ↓
5. 结果页面     → 显示收藏的食谱列表
```

### 交互说明

| 操作 | 效果 |
|------|------|
| 点击上传区域 | 模拟选择照片 |
| 点击「开始识别食材」 | 进入识别流程 |
| 点击食材标签的 × | 删除食材 |
| 输入并点击「添加」 | 添加新食材 |
| 点击「生成食谱推荐」 | 生成食谱 |
| 点击 ✕ 按钮或左滑 | 跳过食谱 |
| 点击 ❤️ 按钮或右滑 | 收藏食谱 |
| 点击食谱卡片 | 查看详情 |
| 点击右上角 ❤️ | 查看收藏 |

## 设计要点

### 视觉设计
- 渐变紫色主题 (#667eea → #764ba2)
- 圆角卡片设计
- 阴影层次感
- 流畅的过渡动画

### 响应式
- 最大宽度 400px（模拟手机）
- 移动端全屏显示
- 触摸友好的按钮尺寸

### 交互反馈
- 上传区域 hover 效果
- 按钮 hover 放大效果
- 卡片滑动动画
- Loading 旋转动画
