# 客户与组织 - 用户故事

## 背景
主数据需要统一维护客户与组织信息，并且通过组织标签驱动客户同步和上下游关系管理，保证组织、客户与关系数据一致。

## 用户故事列表

### Story 1: 创建与维护组织
**作为** 系统管理员
**我想要** 创建、编辑与查询组织基础信息与标签
**以便于** 统一管理各类合作方与组织档案

#### 验收标准
- [ ] GIVEN 组织编码唯一 WHEN 提交创建组织 THEN 系统保存组织并生成组织ID
- [ ] GIVEN 组织标签包含 CUSTOMER WHEN 保存组织 THEN 系统同步创建或更新客户信息
- [ ] GIVEN 组织状态变更 WHEN 将组织置为停用 THEN 系统同步更新客户有效性

### Story 2: 创建与维护客户
**作为** 主数据管理员
**我想要** 创建、编辑与查询客户基础信息
**以便于** 为后续业务配置与关联提供稳定的客户主数据

#### 验收标准
- [ ] GIVEN 客户编码唯一 WHEN 提交创建客户 THEN 系统保存客户并记录基础信息
- [ ] GIVEN 新建客户 WHEN 创建成功 THEN 系统异步创建组织并回写组织ID
- [ ] GIVEN 客户ID存在 WHEN 更新客户 THEN 系统保存更新内容

### Story 3: 维护组织与客户关系
**作为** 主数据管理员
**我想要** 维护客户与组织的关系记录
**以便于** 管理客户与零售商、品牌、供应商等关系

#### 验收标准
- [ ] GIVEN 客户与组织已存在 WHEN 创建关系 THEN 系统保存关系记录
- [ ] GIVEN 关系记录存在 WHEN 删除关系 THEN 系统移除关系记录
- [ ] GIVEN 组织标签明确 WHEN 按标签查询关系 THEN 系统返回匹配关系列表

### Story 4: 查询与分页检索
**作为** 系统管理员
**我想要** 通过条件与分页查询组织与客户
**以便于** 快速定位目标数据并进行维护

#### 验收标准
- [ ] GIVEN 查询条件 WHEN 查询组织 THEN 系统返回匹配组织列表
- [ ] GIVEN 分页条件 WHEN 查询客户 THEN 系统返回分页结果与总数
- [ ] GIVEN 关键字 WHEN 查询客户 THEN 系统按编码或名称模糊匹配
