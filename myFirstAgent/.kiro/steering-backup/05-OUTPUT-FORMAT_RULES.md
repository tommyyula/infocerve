# 输出格式规范

> 优先级：第六优先级
> 
> 目的：确保 AI 输出的内容便于用户复制到记事本等纯文本编辑器

---

## 核心原则

用户可能需要将 AI 的回复复制到记事本、Word、企业微信等工具中，因此输出格式必须兼容纯文本环境。

---

## 禁止使用的格式

1. 禁止使用 Markdown 表格
   - 表格在纯文本环境下会变成乱码
   - 改用列表或缩进格式

2. 禁止使用特殊 Unicode 符号
   - 禁止：箭头符号如 → ← ↑ ↓
   - 禁止：勾选符号如 ✅ ❌ ☑ ☐
   - 禁止：星号符号如 ⭐ ★ ☆
   - 禁止：警告符号如 ⚠️ 🚨 💡
   - 禁止：数字圈如 ① ② ③
   - 改用纯 ASCII 字符

3. 禁止使用 Emoji
   - 禁止：表情符号如 😀 👍 🎯
   - 改用文字描述

---

## 推荐的格式

1. 标题格式
   - 使用纯文本分隔线
   - 示例：
     ```
     一、标题内容
     ---
     ```

2. 列表格式
   - 使用 - 或数字
   - 示例：
     ```
     - 第一项
     - 第二项
     
     1. 第一步
     2. 第二步
     ```

3. 强调格式
   - 使用 [重要] 或 [注意] 等文字标记
   - 示例：
     ```
     [重要] 这是重要内容
     [注意] 这是注意事项
     ```

4. 表格替代格式
   - 使用缩进列表代替表格
   - 示例：
     ```
     模块列表：
     
     L1 接入层
       - 职责：对上游提供统一 Job API
       - 依赖：无
     
     L2 业务编排层
       - 职责：Job/Task 管理、Workflow 引擎
       - 依赖：L1
     ```

5. 分隔线
   - 使用三个或更多的 - 符号
   - 示例：
     ```
     ---
     ```

---

## 示例对比

错误示例（包含特殊符号）：

```
## 🎯 核心功能

| 模块 | 职责 |
|------|------|
| L1 | 接入层 |
| L2 | 编排层 |

✅ 已完成
❌ 未完成
→ 下一步
```

正确示例（纯文本友好）：

```
核心功能
---

模块列表：

L1 接入层
  - 职责：对上游提供统一 Job API

L2 编排层
  - 职责：Job/Task 管理、Workflow 引擎

状态说明：
- [完成] 已完成的任务
- [未完成] 未完成的任务
- [下一步] 接下来要做的事情
```

---

## 适用场景

此规范适用于：
- 需求讨论和确认
- 设计方案说明
- 问题列表和回答
- 任何用户可能需要复制的内容

此规范不适用于：
- 代码文件（代码文件按语言规范）
- Markdown 文档文件（如 requirements.md、design.md）
- 仅在 IDE 中查看的内容
